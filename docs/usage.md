# Splat Replay ユーザーガイド

この文書は、配布版の `SplatReplay.zip` を展開して利用する方向けのガイドです。
初回起動時のセットアップウィザードと、日常利用の流れを中心にまとめています。

ソースコードからのビルドや開発環境構築は対象外です。
開発者向け手順は [開発ガイド](./DEVELOPMENT.md) を参照してください。

## この文書の対象

- GitHub Releases から配布版を取得して使いたい方
- 初回セットアップで何を準備すればよいか知りたい方
- 普段の起動方法や止め方を確認したい方

## 利用開始までの流れ

1. GitHub Releases から `SplatReplay.zip` をダウンロードします。
2. ZIP ファイルを任意のフォルダへ展開します。
3. 展開先の `SplatReplay.exe` を起動します。
4. 初回はセットアップウィザードに従って必要な準備を行います。
5. セットアップ完了後、通常利用を開始します。

## 事前に必要なもの

- Windows 11 の PC
- Nintendo Switch とドック
- HDMI 入力対応、1080p/30fps 以上推奨のキャプチャーボード
- OBS Studio
- YouTube アカウント（Google アカウント）
  - 初回セットアップ時の認証に必要です。
- 必要に応じてマイク
  - 文字起こし機能を使う場合に必要です。
- 必要に応じて Groq API キー
  - 文字起こし機能を使う場合に必要です。

補足:

- OBS Studio、FFmpeg、Tesseract OCR などの外部ソフトは、セットアップウィザードの進行中に確認します。
- 録画・編集・アップロードの詳しい仕様を確認したい場合は [外部設計書](./external_spec.md) を参照してください。

## セットアップウィザード

初回起動時はセットアップウィザードが開きます。
画面の案内に従って、必要な項目を順に確認してください。

### 1. ハードウェア確認

- Switch、キャプチャーボード、PC の接続状態を確認します。
- 必要に応じてマイクの利用有無も確認します。
- YouTube アップロードに必要なネットワーク接続も確認します。

### 2. FFmpeg

- 動画編集に必要な FFmpeg を利用できる状態か確認します。
- 未導入の場合は、画面の案内に従って準備してください。

### 3. OBS Studio

- OBS Studio の導入状況や利用準備を確認します。
- `obs-ndi` と `NDI 6 Runtime`、録画設定、WebSocket のパスワードなど、録画に必要な設定をこの段階で見直します。

### 4. Tesseract OCR

- OCR に必要な Tesseract OCR を利用できる状態か確認します。
- 文字認識に必要なため、この段階で準備してください。

### 5. フォント

- サムネイル生成に必要なフォントを確認します。
- 利用可能なフォントがない場合は、案内に従って準備してください。

### 6. 文字起こし

- 文字起こし機能を使う場合は、マイク名、Groq API キー、言語、認識させたい単語を 1 行ずつ登録する辞書を設定します。
- 文字起こしを使わない場合は、このステップをスキップできます。

### 7. YouTube

- YouTube アップロードに必要な初期設定を行います。
- Google Cloud でプロジェクト作成、YouTube Data API の有効化、OAuth 同意画面と OAuth 認証情報の準備を行います。
- `client_secret.json` を `SplatReplay\config` に配置します。
- 15 分を超える動画を扱う場合は、YouTube 側のアカウント認証を行います。Google Cloud 側の準備とは別です。
- 公開範囲の設定を確認します。
- 実際のアップロード時に YouTube 認証やアクセス許可が必要になると、ブラウザが開きます。

すべてのステップが完了すると、通常の利用に進めます。

## 使い方

### 起動する

1. 展開したフォルダの `SplatReplay.exe` を起動します。
2. OBS Studio を利用できる状態にします。
3. Switch をキャプチャーボード経由で PC に接続します。

### 普段の流れ

- アプリ起動後は、ゲーム映像の監視、録画、編集、アップロードが自動で進みます。
- 文字起こしを有効にしている場合は、その設定に従って処理されます。
- OBS と Switch の接続、必要な認証や設定が済んでいれば、ひとまず通常の処理に進める状態と考えてよいです。
- YouTube 認証やアクセス許可が必要なときはブラウザが開くので、画面表示に従って手続きを完了してください。

### 停止する

- 通常はアプリケーションウィンドウを閉じて終了します。
- 録画・編集・アップロード中は、処理が終わるまで閉じない方が安全です。
- 処理状況によっては、終了まで少し時間がかかる場合があります。

## トラブルシューティング

### セットアップウィザードが完了できない

- 途中で未完了の項目があると、通常利用へ進めません。
- 画面上で強調表示されている項目を確認し、必要な準備を完了してください。

### YouTube 認証が求められる

- YouTube 認証やアクセス許可が必要なときは、ブラウザでの手続きが必要です。
- ブラウザ画面の案内に従って認証を完了してください。

### 録画やプレビューが始まらない

- まず Switch とキャプチャーボードの接続、OBS Studio のソース・デバイス選択を確認してください。
- 次に `obs-ndi`、`NDI 6 Runtime`、録画設定、WebSocket のパスワードを見直してください。
- 画面が出ない場合は、OBS 側のプレビュー有効化も確認してください。

### Switch の電源 OFF を検知できない

- キャプチャーボードによっては電源 OFF 時の画面が異なります。
- Switch を電源 OFF にした状態の画面をキャプチャし、その画像を用意してください。
- 展開したフォルダの `assets/matching/power_off.png` を差し替えて調整してください。

## カスタマイズ

### サムネイル画像を差し替える

1. 1920x1080 の PNG 画像を用意します。
2. 展開したフォルダの `assets/thumbnail/thumbnail_overlay.png` として保存します。

補足:

- 透過部分がある場合は、自動生成サムネイルの上に重ねて表示されます。
- 完全に不透明な画像を置いた場合は、自動生成部分が見えなくなります。
